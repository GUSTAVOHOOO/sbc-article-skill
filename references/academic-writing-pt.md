<voice_and_person>
**Pessoa gramatical:**
- Terceira pessoa do singular: "Este trabalho propõe...", "O método apresentado..."
- Terceira pessoa do plural: "Os autores desenvolveram...", "Os resultados indicam..."
- Primeira pessoa do plural aceita em alguns contextos: "Neste artigo, apresentamos..."
- NUNCA: "Eu fiz...", "A gente observou...", "Na minha opinião..."

**Voz:**
- Voz ativa preferida: "O algoritmo processa 1000 itens/s"
- Voz passiva aceitável para metodologia: "O experimento foi conduzido..."
- Balancear: não use voz passiva em mais de 40% das frases
</voice_and_person>

|----------|--------|

| muitos | diversos / numerosos / N% |
| poucos | reduzido número / apenas X% |
| ótimo/excelente | superior / de alto desempenho |
| ruim/fraco | inferior / de baixo desempenho |
| rápido | eficiente / com menor latência |

| ver | observar / verificar / constatar |
| falar sobre | abordar / discutir / tratar |
| descobrir | identificar / detectar / verificar |
| junto com | em conjunto com / associado a |
| mas | no entanto / contudo / todavia |
| então | portanto / assim / desta forma |

</formal_vocabulary>

<transitions>
**Conectivos por função:**

Adição: "Ademais", "Além disso", "Igualmente", "Da mesma forma"
Contraste: "No entanto", "Contudo", "Todavia", "Por outro lado", "Em contrapartida"
Causa: "Porque", "Uma vez que", "Dado que", "Em razão de"
Consequência: "Portanto", "Assim", "Desta forma", "Consequentemente", "Logo"
Exemplificação: "Por exemplo", "A título de ilustração", "Como ilustração"
Conclusão: "Em síntese", "Conclui-se que", "Diante do exposto"
Sequência: "Inicialmente", "Em seguida", "Posteriormente", "Por fim"
</transitions>

<phrases_to_avoid>
**Frases vagas — substitua:**

| Evitar | Usar em vez |
|--------|------------|
| "É importante ressaltar que..." | Afirme diretamente |
| "Vale a pena mencionar que..." | Afirme diretamente |
| "Podemos observar que..." | "Observa-se que..." |
| "É possível notar que..." | "Nota-se que..." |
| "De certa forma..." | Seja específico |
| "Em algum momento..." | Especifique quando |
| "Obviamente..." | Nunca — se é óbvio, não diga |
| "Claramente..." | Mostre, não declare |
| "Sem dúvida..." | Prove com dados |
| "Revolucionário" | Descreva o que muda especificamente |
</phrases_to_avoid>

<acronyms>
**Regra de siglas:**
- Primeira ocorrência: nome completo seguido da sigla entre parênteses
  - Ex: "Inteligência Artificial (IA)"
  - Ex: "Natural Language Processing (NLP)"
- Ocorrências subsequentes: apenas a sigla
- No abstract e no resumo: definir separadamente (são seções independentes)
- Título: evitar siglas (exceto se a sigla for mais conhecida que o nome completo)
</acronyms>

<paragraph_structure>
**Estrutura de parágrafo acadêmico:**

1. **Tópico-frase** — afirmação principal do parágrafo (1ª frase)
2. **Desenvolvimento** — evidências, dados, explicação (2-5 frases)
3. **Conclusão/transição** — síntese ou link para próximo parágrafo (última frase)

Tamanho ideal: 4-8 linhas no formato SBC (Times 12pt, coluna única)
Máximo: 12 linhas — se maior, divida em dois parágrafos

**Exemplo de parágrafo bem estruturado:**
> "A abordagem proposta supera os métodos tradicionais em termos de eficiência temporal. [TÓPICO] Nos experimentos realizados, o tempo médio de processamento foi de 0,3s por consulta, enquanto os métodos de referência BERT e RoBERTa apresentaram tempos de 1,2s e 1,5s, respectivamente [SILVA, 2023]. [DADOS] Essa diferença de 4x na velocidade viabiliza a aplicação em cenários de tempo real, como sistemas de recomendação em produção. [CONCLUSÃO/RELEVÂNCIA]"
</paragraph_structure>

<numbers_and_data>
**Apresentação de números:**
- Escrever por extenso de 1 a 9: "três experimentos", "oito participantes"
- Algarismos a partir de 10: "15 iterações", "100 amostras"
- Porcentagem: "95%" (não "95 por cento" no texto corrido)
- Decimais: usar vírgula em PT, ponto em EN — "0,95" (PT) / "0.95" (EN)
- Sempre contextualize: "acurácia de 0,95" → "acurácia de 95%, superior à baseline de 87%"
</numbers_and_data>
