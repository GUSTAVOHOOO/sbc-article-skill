<article_structure>

<imrad_sbc>
Artigos SBC seguem estrutura IMRaD adaptada:
**I**ntrodução → Trabalhos **R**elacionados → **M**etodologia → Result**a**dos → Conclusão

Variações aceitas:
- "Proposta" ou "Desenvolvimento" no lugar de "Metodologia" (sistemas/engenharia)
- "Avaliação" ou "Experimentos" no lugar de "Resultados"
- "Trabalhos Relacionados" embutida na Introdução (artigos curtos)
- "Discussão" como seção separada após Resultados (artigos experimentais)
</imrad_sbc>

<section_purposes>
**Propósito de cada seção:**

**Introdução** — responde: "Por que este trabalho existe?"
- Contextualiza o problema no campo
- Identifica o gap (o que ainda não foi resolvido)
- Enuncia a contribuição claramente
- Indica a estrutura do artigo

**Trabalhos Relacionados** — responde: "O que já existe e como este trabalho se diferencia?"
- Não é uma lista de artigos — é uma análise crítica
- Agrupa trabalhos por abordagem
- Para cada grupo: descreve, cita, aponta limitação
- Fecha mostrando como este trabalho supera as limitações

**Metodologia** — responde: "Como foi feito? (de forma reprodutível)"
- Descreve dataset, ferramentas, parâmetros
- Justifica escolhas com citações
- Permite replicação por outro pesquisador

**Resultados** — responde: "O que foi encontrado?"
- Apresenta dados antes de interpretá-los
- Compara com baseline ou trabalhos relacionados
- Usa tabelas (comparações numéricas) e figuras (tendências)
- Responde ao objetivo enunciado na introdução

**Conclusão** — responde: "O que foi alcançado e o que vem depois?"
- Síntese do que foi feito (não repete resultados em detalhe)
- Limitações do trabalho (honestidade científica)
- Trabalhos futuros (concretos, não genéricos)
</section_purposes>

<contribution_types>
**Tipos de contribuição aceitos na SBC:**

1. **Proposta de método/algoritmo** — novo approach para problema conhecido
2. **Aplicação em novo domínio** — método existente aplicado a novo contexto
3. **Estudo experimental/comparativo** — comparação sistemática de abordagens
4. **Survey/Mapeamento sistemático** — visão geral da área
5. **Ferramenta/Sistema** — artefato de software com avaliação
6. **Estudo de caso** — aplicação real com análise detalhada
7. **Replicação** — reprodução de estudo para validar resultados

Cada tipo tem ênfase diferente na estrutura:
- Proposta de método: metodologia + resultados mais longos
- Survey: trabalhos relacionados é o corpo principal
- Estudo de caso: contexto e análise qualitativa mais extensos
</contribution_types>

<related_work_patterns>
**Padrões para Trabalhos Relacionados:**

**Padrão por categoria:**
> "Abordagens baseadas em [técnica A] [CITA, CITA] obtêm [resultado], porém [limitação]. Métodos que utilizam [técnica B] [CITA] superam [limitação anterior], mas sofrem com [nova limitação]. O presente trabalho difere ao [diferencial específico]."

**Padrão cronológico (usar apenas para áreas novas):**
> "Os primeiros trabalhos na área [CITA] estabeleceram [base]. Posteriormente, [CITA] aprimorou [aspecto]. Recentemente, [CITA] incorporou [inovação]. Ainda assim, [gap que este trabalho aborda]."

**Tabela comparativa (opcional, eficaz para 5+ trabalhos):**

| Trabalho | Abordagem | Dataset | Métrica | Limitação |
|----------|-----------|---------|---------|-----------|
| [CITA] | X | Y | Z | W |
| Este trabalho | ... | ... | ... | — |
</related_work_patterns>

<introduction_formula>
**Fórmula da Introdução (4-5 parágrafos):**

P1 — Contexto amplo (2-3 frases): "O campo de X cresceu significativamente... [dados/CITA]"
P2 — Problema específico (3-4 frases): "Apesar do avanço, [problema específico] permanece um desafio... [CITA demonstrando o gap]"
P3 — Motivação/Impacto (2-3 frases): "Resolver [problema] é crucial porque [impacto concreto]..."
P4 — Contribuição (lista ou parágrafo): "Este trabalho apresenta as seguintes contribuições: (i) ...; (ii) ...; (iii) ..."
P5 — Organização: "O restante deste artigo está organizado como segue. A Seção 2 apresenta os trabalhos relacionados. A Seção 3 descreve a metodologia proposta. ..."
</introduction_formula>
