<process>
## Passo 1: Identificar seção e contexto

Pergunte (se não informado):
- Qual seção? (abstract, introdução, trabalhos relacionados, metodologia, resultados, conclusão)
- O usuário tem rascunho para melhorar, ou quer escrever do zero?
- Qual o tema/contribuição central do artigo?

## Passo 2: Aplicar estrutura da seção

### Abstract (Resumo)
Estrutura obrigatória em 150-200 palavras:
1. **Contexto/Problema** (1-2 frases): Por que este problema importa?
2. **Gap** (1 frase): O que ainda não foi resolvido?
3. **Proposta** (1-2 frases): O que este trabalho faz?
4. **Método** (1 frase): Como foi feito?
5. **Resultado principal** (1-2 frases): O que foi encontrado/alcançado?
6. **Conclusão/Impacto** (1 frase): Qual a relevância?

### Introdução
Estrutura em 4-6 parágrafos:
1. **Contextualização ampla** — importância do tema
2. **Problema específico** — o que ainda não está resolvido
3. **Motivação** — por que resolver agora
4. **Contribuição** — o que este trabalho propõe (liste pontos com bullet se necessário)
5. **Organização do artigo** — "O restante deste artigo está organizado como segue..."

### Trabalhos Relacionados
- Agrupe trabalhos por abordagem/categoria, não liste cronologicamente
- Para cada grupo: descreva a abordagem, cite os trabalhos, aponte limitação
- Termine com parágrafo mostrando como este trabalho é diferente/superior
- Mínimo 8-10 referências relevantes

### Metodologia
- Descreva de forma **reprodutível**: outro pesquisador deve conseguir repetir
- Inclua: dataset, ferramentas, parâmetros, critérios de avaliação
- Use subseções se a metodologia for complexa
- Justifique escolhas metodológicas com citações

### Resultados
- Apresente resultados antes de interpretá-los
- Use tabelas para comparações numéricas (ambiente LaTeX: `table` + `tabular`)
- Use figuras para tendências e distribuições
- Compare com baselines ou trabalhos relacionados
- Responda à hipótese/objetivo proposto na introdução

### Conclusão
Estrutura em 3-4 parágrafos:
1. **Síntese** — o que foi feito e alcançado (sem repetir resultados detalhados)
2. **Limitações** — honesto sobre o que o trabalho não cobre
3. **Trabalhos futuros** — direções concretas e específicas
4. **Impacto** (opcional) — contribuição para a área

## Passo 3: Escrever com estilo acadêmico

Regras obrigatórias:
- Terceira pessoa: "Este trabalho propõe...", "Os autores apresentam...", "Foi realizado..."
- Conectivos formais: "Ademais", "Portanto", "Desta forma", "No entanto", "Contudo"
- Evitar: "muito", "bastante", "incrível", "revolucionário" — seja preciso e neutro
- Toda afirmação factual → citação
- Siglas: defina na primeira ocorrência — "Processamento de Linguagem Natural (PLN)"

## Passo 4: Usar templates de seção

Consulte `templates/section-templates.md` para a seção solicitada:
- Use a estrutura de parágrafos sugerida como scaffolding
- Adapte as frases-modelo ao conteúdo específico do usuário
- Respeite os tamanhos esperados por seção

## Passo 5: Verificar coesão e qualidade

Após escrever, aplique a mini-checklist de qualidade:
- A seção responde à pergunta que ela deve responder?
- Há transições adequadas entre parágrafos?
- As afirmações têm suporte (citações ou dados)?
- O tamanho está dentro do esperado para o evento?
- Score D6 (escrita) desta seção seria ≥4/5? (sem coloquialismos, terceira pessoa, voz ativa >60%)
</process>

- [ ] Seção segue estrutura específica para seu tipo
- [ ] Escrita em terceira pessoa, linguagem formal
- [ ] Afirmações com citações ou dados
- [ ] Conectivos formais usados corretamente
- [ ] Tamanho adequado ao padrão SBC
</success_criteria>
